Assignment 3: JS DOM

The most challenging part of this assignment was updating the filter results to remove the posts in the DOM that did not match the search results. I was repeatedly getting the following error when I was attempting to filter the items in the DOM: "index.js:7 Uncaught TypeError: filterInputs.getElementById is not a function". I solved this problem by directly accessing the filter-text ID element, instead of accessing it through the filter-input-container first, as I was previously attempting to access the inputs as an array, without specifying the array element.

I had previously not known about how to access parts of the DOM and write elements into it. I was familiar with how to make the hidden modal appear on-click of the Add post button and how to set an event listener. This assignment allowed me to practice using the DOM manipulation and event handling concepts that I had learned in lecture.

The class Discord server was very useful for me in solving the problem specified above, as it allowed me to get a second opinion on why this error was occurring, which ended up being a simple fix. Assignment 2: Matching CSS to mocks.

This assignment was practice for using CSS to style a template HTML page to match given specifications/mocks. In the process of styling this page, I faced a few challenges:

Header: Initially, my main challenge was to format the header. After sucessfully shifting the page title to be on the same line as the Benny Beaver picture, I had trouble with properly positioning the text to match the mocks. By positioning the text to the bottom of the header and adding padding, I was able to shift its location.

Filter box: I initially attempted to format the filter box's fields using the flex box method. However, I found that the styling was much easier to acheive by manually changing the float positions and inner box size/padding/margins.

Result Formatting: The most challenging part of the assignment for me was formatting the search results. I attempted to use flex boxes in order to easily space the results, and have them wrap when the viewport shrinks/grows. However, despite the margins and padding on the boxes being the same, I was unable to get them to neatly align in a grid format. I attempted to use various forms of formatting these search results (besides using flex boxes), but the only result that got me the closest to the mocks was by using flex-boxes and altering the margins/padding.

In this assignment, I learned CSS syntax and how to apply to create a styled webpage based on generic HTML. Previously, I had not used much CSS, so this assignment allowed me to practically apply the new syntax and concepts (such as flex boxes) that I had learned in class, and may not have fully understood.

The resources that helped me the most were the Mozilla CSS documentation, W3 Schools CSS Web tutorials, and the class lecture notes on CSS. These resources helped me learn the CSS syntax necessary to style the webpage. For a specific problem relating to the header positioning, I was able to get help from fellow students on the class Discord server.

For this assignment, I had to Google the CSS modifiers for specific HTML attributes (such as modifying the "position" property), so that I could properly define the styles that I needed. I also had to Google tutorials for CSS3 Flexboxs, to learn how to properly style flexboxes.
